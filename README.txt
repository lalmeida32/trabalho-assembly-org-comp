Conversor de números binários, decimais e hexadecimais

Erros tratados:
- O número deve caber em 32 bits, unsigned (no máximo 4.294.967.295)
- A entrada não deve ser vazia
- A entrada não deve superar 32 caracteres
- Números binários devem incluir apenas caracteres 0-1
- Números decimais devem incluir apenas caracteres 0-9
- Números hexadecimais devem incluir apenas caracteres 0-9a-f


***IMPORTANTE***

Para executar o código no MARS, ativar a caixa "assemble all files in directory", encontrada dentro da aba "Settings", no canto superior esquerdo.
Além disso, o arquivo main.asm deve ser o ponto de entrada do código.
Ademais, os códigos estão codificados pelo padrão ANSI (Unix LF), e não UTF-8 (Windows CRLF). Foi verificado que o MARS gera arquivos com essa codificação por padrão.
